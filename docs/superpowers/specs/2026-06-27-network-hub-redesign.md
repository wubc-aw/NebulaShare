# 网络枢纽节点选择重构设计

**日期**: 2026-06-27  
**主题**: network-hub 节点选择器重构  
**状态**: 待实现

## 背景与问题

当前 `network-hub.tsx` 的节点选择区域存在以下问题：

1. **顶部“当前节点”显示错误**：代码显示 `groups[0].now`，但后端把 `GLOBAL` 排在最后，所以顶部实际显示的是按字母排序第一个非 GLOBAL 分组（如 AI 分组）的节点，造成“GLOBAL=direct 但顶部显示日本 03”的误导。
2. **订阅商分组对用户无意义**：AI / APPLE / 自动选择 等分组来自订阅配置，作为 TUN 网关用户难以理解也不想日常看到。
3. **缺少生效验证**：用户无法直观看到真实流量到底走了哪条链路、是否按订阅商的分流规则生效。

## 目标

- 一眼看到 TUN 当前真正生效的代理节点。
- 一键切换节点，界面只显示用户关心的“所有可用节点”，按大洲折叠。
- 订阅商的分组配置默认隐藏，需要时再展开。
- 提供实时链路日志，验证分流是否生效。

## 设计方案

### 1. 生效节点卡片（顶部）

位置：网关控制页最上方，替代现有的“当前节点”卡片。

显示内容：
- 当前模式：直连 / 规则分流 / 全局代理
- 生效节点名：
  - 直连模式 → “直连”
  - 规则/全局模式 → `GLOBAL` 分组当前选中的节点
- 节点延迟（最近一次测速结果）
- 点击卡片 → 滚动/展开下方的节点选择器

数据来源：
- `/api/mihomo/status` 获取 `mode`
- `/api/mihomo/groups` 中找到 `name === "GLOBAL"` 的 `now` 字段

### 2. 节点选择器（平面列表，按大洲折叠）

位置：生效节点卡片下方。

行为：
- 调用 `/api/mihomo/groups`，收集所有分组中的真实叶节点，按节点名去重。
- 按大洲分组：亚洲、欧洲、北美洲、南美洲、大洋洲、非洲、其他。
- 每个大洲区块可折叠。
- 每个节点显示：名称、延迟（颜色：绿 <100ms / 黄 <200ms / 红 ≥200ms / 灰色未测）、类型标记（可选）。
- 点击节点 → 调用 `/api/mihomo/select`，设置 `group="GLOBAL"`。
- 顶部提供“刷新延迟”按钮 → 调用 `/api/mihomo/test/group/GLOBAL` 触发测速，然后重新拉取 `/api/mihomo/groups` 更新延迟。

地区识别规则（根据节点名关键词）：

| 大洲 | 关键词示例 |
|------|-----------|
| 亚洲 | 香港、台湾、日本、新加坡、韩国、泰国、马来西亚、越南、印度、菲律宾、印尼、中国、上海、北京、澳门、HK、TW、JP、SG、KR、TH、MY、VN、IN、PH、ID |
| 欧洲 | 英国、德国、法国、意大利、荷兰、西班牙、瑞士、瑞典、俄罗斯、波兰、土耳其、芬兰、挪威、丹麦、比利时、奥地利、爱尔兰、葡萄牙、希腊、UK、GB、DE、FR、IT、NL、ES、CH、SE、RU、PL、TR、FI、NO、DK |
| 北美洲 | 美国、加拿大、美國、USA、US、CA、Canada |
| 南美洲 | 巴西、阿根廷、智利、秘鲁、哥伦比亚、BR、AR、CL、PE、CO |
| 大洋洲 | 澳大利亚、新西兰、澳洲、AU、NZ、Australia、New Zealand |

### 3. 订阅商配置（默认折叠）

位置：节点选择器下方。

行为：
- 标题为“订阅商配置”。
- 默认折叠，只显示标题行和分组数量。
- 展开后显示所有原始 selector 分组（AI、APPLE、自动选择、Fallback 等）。
- 每个分组以 chip 列表展示成员，当前选中的高亮。
- 点击 chip → 调用 `/api/mihomo/select` 切换该分组的当前节点。

### 4. 链路日志（最底部）

位置：订阅商配置下方。

行为：
- 轮询 `/api/mihomo/connections/recent`（建议 3 秒一次）。
- 每行显示：时间、客户端 IP、目标域名/IP、匹配规则、实际链路链（如 `AI → 美国 01` 或 `GLOBAL → 日本 03`）。
- 去重逻辑：以 **客户端 IP + 目标域名/IP** 为键，只保留最新一条记录；只有当链路链发生变化时才更新该行。
- 表格顶部显示当前记录数，并提供“清空”按钮（仅清空前端展示）。

用途：让用户直接验证 ChatGPT 是否走了 AI 分组、Apple 服务是否走了 APPLE 分组、其他流量是否走了 GLOBAL。

## 错误处理

- `/api/mihomo/groups` 失败时：生效节点卡片显示错误状态，节点选择器区域显示重试按钮。
- `/api/mihomo/select` 失败时：弹出 toast / 行内错误，不更新选中状态。
- `/api/mihomo/connections/recent` 失败时：链路日志暂停轮询并显示错误提示，可手动重试。

## 数据流

```
生效节点卡片
  ← /api/mihomo/status (mode)
  ← /api/mihomo/groups (GLOBAL.now, GLOBAL.members)

节点选择器
  ← /api/mihomo/groups (所有分组的 members，去重后按大洲分组)
  → /api/mihomo/select (设置 GLOBAL)
  → /api/mihomo/test/group/GLOBAL (刷新延迟)

订阅商配置
  ← /api/mihomo/groups (所有 selector 分组)
  → /api/mihomo/select (设置指定分组)

链路日志
  ← /api/mihomo/connections/recent (轮询)
```

## 关键文件

- `nebula-share-frontend/app/globals.css`：可能需要新增/调整节点选择器和链路日志的样式。

## 边界情况

- 节点名同时匹配多个大洲关键词时，按亚洲、欧洲、北美洲、大洋洲、南美洲、非洲的优先级取第一个匹配。
- 节点延迟为 0 或未测时显示“--”，不显示 0ms。
- `GLOBAL` 分组不存在时，生效节点卡片显示“未配置”。
- 直连模式下，节点选择器仍可操作；点击节点后自动切换为“规则分流”模式，并把该节点设为 GLOBAL 当前选择，确保节点立即生效。
